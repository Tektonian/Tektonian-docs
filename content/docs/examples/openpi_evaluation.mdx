## What This Tutorial Does

This tutorial evaluates RoboCasa OpenPI checkpoints on Simulac with one vectorized evaluator:

- [`eval_robocasa_openpi.py`](https://github.com/Tektonian/Simulac/blob/main/docs/publish/examples/eval/eval_robocasa_openpi.py)

You will:

1. Create an isolated example venv.
2. Check OpenPI imports and Simulac access.
3. Inspect one RoboCasa observation.
4. Get the example evaluator script.
5. Run a short evaluation.
6. Run the full pi0 evaluation.
7. Run the full pi0.5 evaluation.
8. Review the observation and action mapping.

Outputs are written to:

```text
.logs/
```

## 3. Smoke Test One Observation

This verifies that Simulac returns the image keys and image format expected by the examples.

```python
import numpy as np
from simulac.gym_style import get_env_list, init_bench

benchmark_id = "Tektonian/RobocasaPre"
env_id = get_env_list(benchmark_id)[0]

env = init_bench(benchmark_id, env_id)
try:
    obs, info = env.reset(0)
    print("ENV_ID: ", env_id)
    print("Task description: ", info["task_description"])
    for camera, image in obs["images"].items():
        image = np.asarray(image)
        print(camera, image.shape, image.dtype)
finally:
    env.close()
```

Expected output:

```text
ENV_ID: Atomic/Seen/CloseBlenderLid
Task description:  Close the lid blender by securely placing the lid on top.cam_0_rgb (256, 256, 3) int64
cam_1_rgb (256, 256, 3) int64
cam_2_rgb (256, 256, 3) int64
```

## 4. Smoke Test Parallel Environment

This checks image keys and formats using parallel execution: it creates `num_envs` environments with `make_vec()` and resets each one with one seed (`[0] * num_envs`).

```python
import numpy as np
from simulac.gym_style import get_env_list, init_bench, make_vec

benchmark_id = "Tektonian/RobocasaPre"
env_path = get_env_list(benchmark_id)[0]
env_id = env_path
num_envs = 2

envs = [init_bench(benchmark_id, env_id) for _ in range(num_envs)]
vec_env = make_vec(envs)

try:
    outputs = vec_env.reset([0] * num_envs)
    for _idx, (obs, info) in enumerate(outputs):
      print("Env ", _idx)
      print("ENV_ID: ", env_id)
      print("Task description: ", info["task_description"])
      for camera, image in obs["images"].items():
          image = np.asarray(image)
          print(camera, image.shape, image.dtype)
finally:
    vec_env.close()
```

Expected output:

```text
Env  0
ENV_ID:  Atomic/Seen/CloseBlenderLid
Task description:  Close the lid blender by securely placing the lid on top.
cam_0_rgb (256, 256, 3) int64
cam_1_rgb (256, 256, 3) int64
cam_2_rgb (256, 256, 3) int64
Env  1
ENV_ID:  Atomic/Seen/CloseBlenderLid
Task description:  Close the lid blender by securely placing the lid on top.
cam_0_rgb (256, 256, 3) int64
cam_1_rgb (256, 256, 3) int64
cam_2_rgb (256, 256, 3) int64
```

The examples convert these images to contiguous `uint8` RGB arrays before passing them to OpenPI or OpenCV.

## 5. Get the Example Script

Use the RoboCasa OpenPI evaluator from the Simulac repository:

- [`eval_robocasa_openpi.py`](https://github.com/Tektonian/Simulac/docs/publish/examples/eval/eval_robocasa_openpi.py)

Download or copy this script into your working directory before running the commands below.

## 6. Run a Short Evaluation

Before a full evaluation, run one episode per task with one environment copy at a time.

```bash
python eval_robocasa_openpi.py \
  --model-name pi0 \
  --episodes-per-env 1 \
  --max-vec-envs 1
```

The first OpenPI policy call can be slow because JAX compiles the model.

## 7. Run the pi0 Evaluation

```bash
python eval_robocasa_openpi.py \
  --model-name pi0 \
  --episodes-per-env 50 \
  --max-vec-envs 2
```

The evaluator:

1. Loads the OpenPI policy.
2. Converts Simulac observations to OpenPI inputs.
3. Runs seeded task copies in vectorized batches.
4. Saves one JSONL result per task.
5. Saves MP4 videos from `cam_0_rgb`.

Typical output files:

```text
.logs/robocasa_pi0_benchmark_<RUN_ID>.log
.logs/robocasa_pi0_benchmark_results_<RUN_ID>.jsonl
.logs/robocasa_pi0_benchmark_videos_<RUN_ID>/*.mp4
```

Use the videos first when debugging policy behavior.

## 9. Observation and Action Mapping

Simulac provides observation data through:

- `obs["images"]["cam_0_rgb"]`
- `obs["images"]["cam_2_rgb"]`
- robot state values in `obs["states"]`
- the language prompt in `info["task_description"]`

OpenPI receives:

- `observation/image` from `cam_0_rgb`
- `observation/wrist_image` from `cam_2_rgb`
- `observation/state` from concatenated robot state
- `prompt` from `info["task_description"]`

The examples convert Simulac images to contiguous `uint8` RGB arrays before model inference and video writing.

Keep the action dimensions separate:

- `ACTION_DIM = 12`: action vector sent to Simulac.
- `MODEL_ACTION_DIM = 32`: padded OpenPI model action width.

OpenPI predicts using the padded model action width. The evaluator converts the model output into the 12-dimensional Simulac action vector before calling `env.step(...)`.
