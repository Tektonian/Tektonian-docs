---

title: "Run Benchmarks in Parallel"
description: "Step multiple environments at once with make_vec()."
------------------------------------------------------------------

Run multiple benchmark environments in parallel with a single API to increase throughput and scale experiments easily.

`make_vec()` wraps multiple environments and lets you step them together, making it easy to scale from one environment to many.

---

## Step 1: build a vector environment

```python
from simulac.gym_style import init_bench, make_vec

envs = [
    init_bench(
        "Tektonian/Metaworld",
        "assembly-v3",
        0,
        {}
    )
    for _ in range(3)
]

vec_env = make_vec(envs)
```

---

## Step 2: reset a vector environment

```python
seeds = [0, 1, 2]
results = vec_env.reset(seeds)
```

* Provide one seed per environment
* Resets are executed in parallel

---

## Step 3: step all environments at once

```python
results = vec_env.step([[0.0] * 7 for _ in envs])
print(len(results))  # 3
```

* One action per environment (list length must match number of environments)
* Results are returned in the same order as the input list

---

## Step 4: run a full benchmark in parallel

You can run all tasks in a benchmark as a single vectorized batch.

```python
from simulac.gym_style import init_bench, make_vec, get_env_list

# Get all task IDs in the benchmark
env_ids = get_env_list("Tektonian/Metaworld")
print(len(env_ids)) # 50

# Initialize environments
envs = [
    init_bench(
        "Tektonian/Metaworld",
        env_id,
        0,
        {},
    )
    for env_id in env_ids
]

vec_env = make_vec(envs)

n_envs = len(envs)
print(f"Initialized {n_envs} environments.")

# Reset and step
vec_env.reset([0] * n_envs)
results = vec_env.step([[0.0] * 4 for _ in range(n_envs)])
```

Make sure:

* action dimensions match each environment
* action ordering aligns with the environment list

---

## Step 5: mix different benchmarks

You can mix completely different benchmark types in a single vector environment.

```python
from simulac.gym_style import init_bench, make_vec

libero_env = init_bench(
    "Tektonian/Libero",
    "libero_90/KITCHEN_SCENE2_put_the_black_bowl_at_the_back_on_the_plate",
    0,
    {"control_mode": "OSC_POSE"},
)

robomimic_env = init_bench(
    "Tektonian/Robomimic",
    "PickPlaceCan",
    0,
)

vec_env = make_vec([libero_env, robomimic_env])
```

Make sure:

* action dimensions match each environment
* action ordering aligns with the environment list

---

## Notes

* `make_vec()` is useful for scaling experiments and increasing throughput
* You can treat it as a drop-in extension of the single-environment API
* Parallel execution is handled internally (no async code required)
