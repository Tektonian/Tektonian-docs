---

title: "Run Benchmarks in Parallel"
description: "Step multiple environments at once with make_vec()."
------------------------------------------------------------------

Run multiple benchmark environments in parallel with a single API.

`make_vec()` wraps multiple environments and lets you step them together, making it easy to scale from one environment to many.

---

## Step1: build a vector environment

```python
from simulac.gym_style import init_bench, make_vec

envs = [
    init_bench(
        "Tektonian/Libero",
        "libero_90/KITCHEN_SCENE2_put_the_black_bowl_at_the_back_on_the_plate",
        0,
        {"control_mode": "OSC_POSE"},
    )
    for _ in range(3)
]

vec_env = make_vec(envs)
```

---

## Step2: step all environments at once

```python
results = vec_env.step([[0.0] * 7 for _ in envs])
print(len(results))  # 3
```

* One action per environment
* Results are returned in the same order as the input list

---

## Step3: Reset multiple environments

```python
seeds = [0, 1, 2]
results = vec_env.reset(seeds)
```

* Provide one seed per environment
* Resets are executed in parallel

---

## Step4: Mix different benchmarks

You can run different benchmark types in the same vector environment.

```python
from simulac.gym_style import init_bench, make_vec

libero_env = init_bench(
    "Tektonian/Libero",
    "libero_90/KITCHEN_SCENE2_put_the_black_bowl_at_the_back_on_the_plate",
    0,
    {"control_mode": "OSC_POSE"},
)

robomimic_env = init_bench(
    "Tektonian/Robomimic",
    "PickPlaceCan",
    0,
)

vec_env = make_vec([libero_env, robomimic_env])
```

Make sure:

* action dimensions match each environment
* action ordering aligns with the environment list

---

## Notes

* `make_vec()` is useful for scaling experiments and increasing throughput
* You can treat it as a drop-in extension of the single-environment API
